# Git WorkFlow
- master
- develop
    - jjh_calender
    - hgm_node
    - ...
이런식으로
master 브랜치는 릴리즈 버전 보관
develop 브랜치로 업데이트
토픽별 브랜치 생성해서 Merge

- Wiki 에다가 구현/설계 문서 정리
- Issues 에다가 작업 관리/ 마일스톤 설정

- 자주 commit 하기 / git pull 잘하기
- Git 커밋 메시지 영어로 작성하자
- 최대한 파일 겹치지 않게 편집할 것
